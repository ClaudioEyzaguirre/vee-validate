Delayed Validation (Debounced)

Validate Data Values

The basic approach relies on listening to the input or the change events depending on the file type. However most of the time, your values are bound to your Vue instance and some code may change their inputs programatically, the basic approach won't detect this change.

The Solution: The v-validate directive can take an expression, the expression is the data value you wish to validate for example: <input v-validate="email" rules="required|email" type="text" name="email"> Whenever the expression is updated, the validator will validate the new expression value automatically.

Note: The plugin won't retrive the value from the input anymore, it will validate whatever value in your vue instance data. And as you noticed, you don't need to provide a name attribute the expression name will be used instead.
